Introduction

Ultimate Survey is a powerful application that allows non-technical users to create and take complex surveys. In addition, it includes an extremely powerful reporting interface that allows for detailed analysis of survey responses.

All ASP source code is included so that it can be easily modified. Any ASP developer will be able to easily understand the code and make changes to it as needed. All coding was done in accordance with a robust set of coding standards, so the code is clean, consistent, and well documented. The reason for doing this is so that the purchaser can easily install, modify, maintain, and augment the existing code base.

To use this system, your web site must be running Microsoft IIS with full support for Active Server Pages. The application has been written to run against either a SQL Server or MS Access database.

Installation

The steps to install Ultimate Survey are as follows:

  1. Web Server Setup

    a) Installing the ASP files. Unzip the files to a directory on the web server. If necessary, create a virtual directory to point to the application’s root directory. Note: this document assumes that it will be installed in a directory called "UltimateSurvey" but you may change it.

    The directory structure for the application is:

    • UltimateSurvey – Contains .asp files that display allow for creating, taking, and reporting on surveys.

      • Include – Contains all include files (_inc.asp extension)

        • Stylesheet – Contains stylesheet (CSS) for application.

      • Images – Contains all images

        •  Upload - Folder to hold all uploaded images

      • Help – Contains all help files

      • Database – Contains the SQL script needed to install the application on a Microsoft SQL Server database and the MS Access databases

    b) Setting permissions. You will have to set "write" permissions on the folder that will contain your uploaded images (Images/Upload by default) , so that you can upload images for surveys as desired


     

  2. Database Setup

    At this point, you must decide whether to use SQL Server or MS Access.  Choose your database and follow the corresponding instructions below: 

    SQL Server is highly recommended for maximum speed and scalability.

    INTRUCTIONS FOR MS ACCESS

    The database, which has been fully populated with a username and various needed values, is located in the "Database" directory. This folder contains two copies of the Access database. The version named UltimateSurvey.mdb is in Access 2000 format, and the version named UltimateSurvey-97.mdb is in Access 1997 format. Our recommendation is to use the version of the database in Access 2000 format, but both versions should work fine. There are no database passwords by default.

    After deciding which version of Access you will be using, you may delete the other Access database if you like.  At this point, you may want to move the database to another desired location on the server.

    NOTE: The anonymous IIS user must have WRITE permission to this file.


    INSTRUCTIONS FOR SQL SERVER

    a) At this point, you may want to optionally create a new database for the application. If you decide not to create a new database and to use an existing database instead, note that all table names have the prefix "usd_" so that they can easily be distinguished from other tables in the database.

    b) Load the tables into the database. The script for the tables is located in the file, "UltimateSurveyTables.sql," which is located in the "Database" directory. The best way to load the tables is to open Microsoft Query Analyzer, open this file from Query Analyzer, and execute.

    c) Create a user in the database for the Ultimate Survey application. Grant this user full access to all of the tables.

    d) Load the required data into the database. The script for the data is located in the file, "UltimateSurveyData.sql," which is located in the "Database" directory. The best way to load the meta-data is to open Microsoft Query Analyzer, open this file from Query Analyzer, and execute.
     

  3. Database Connection Setup

    The application uses a DSN-less connection. The connection string is located in the file, "SurveyConnection_inc.asp" in the Include directory. This file contains two versions of the constant DB_CONNECTION. One of the versions is for connecting to a SQL Server database, and the other is for using a MS Access database. Above each constant are detailed comments for how to configure the connection string for your database. At this point, comment out the constant that you will not be using, and set up the connection string, as shown.

    In the same file, "SurveyConnection_inc.asp," there is a constant called DATABASE_TYPE. Depending on whether you are using SQL Server or MS Access, the value for this constant must be updated to either "SQLServer" or "MSAccess", as appropriate. These values are noted in the comment above the constant.
     

  4. Getting Started with Surveys

    a) By default, the application has a single user.  The username for this default user is "Admin" and the password is also "Admin". This account has been granted full user rights to the entire application.  Go to the main page of the application and click "Login" to begin using Ultimate Survey.

    b) It is highly recommended that you immediately edit user information by clicking the username in the top right corner of the screen.

    c) Click "Settings" and carefully review each setting. Information on each setting is found in the help section of the application. These settings may be changed by any system administrator at any time.

    d) Add a survey. To start adding surveys, click the "Manage Surveys" tab and then click "Add New Survey"

    e) All actions are explained throughout the application and the help section.

Application Design Notes

This section contains information about various aspects of the application. The purpose in giving this information is twofold: first, with this information, users of the application will be able to better understand the existing functionality. Second, this information will enable administrators of the application and developers to more easily modify the code to their own custom solution.

Initially, the administrator is the only user who can create and edit surveys. As users register, the  administrator can go to "Manage Users" and specify which users are allowed to create surveys, and can even grant administrator privileges to users.

All constants besides the connection string and database type have been written in capital letters, with a prefix of SV (Survey) or USD(Ultimate Software Designs), and are included in /Include/Constants_inc.asp. For example, 'SV_DEFAULT_COMPLETION_MESSAGE'

All functions that are in include files are prefixed with the name of the include file.  For example, if you see the function "surveyCreation_createSurvey", you will know that it is located in "Include/surveyCreation_inc.asp".

For some surveys, and for all administration tasks, a user must first log in. Once logged in, the user's username is stored in a cookie or session, and security is done through validation of this cookie or session.
