File Management: mrTables

mrTables handles the DimensionNet file management differently from the mrInterview activities, because mrTables always opens the data files in read-only mode. (When you create new variables in mrTables, they are stored separately from the main project data.)

The default File Management behavior for mrTables data files is as follows:

• When mrTables starts, it checks the user's folder for the files it needs. If the files exist, these are the ones that mrTables uses.

• If the files do not exist in the user's folder, mrTables checks for them in the shared folder. If the files exist, these are the ones that mrTables uses. Unlike the mrInterview activities, mrTables does not copy the files into the user's folder.

• If the files exist in neither location, you will get an error.

For information about how mrTables manages Tables Document (.mtd) files, see the mrTables documentation.
