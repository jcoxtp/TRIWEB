Uploading Files Created with mrInterview Desktop Applications

The Files activity allows you to upload any project files of your choice into the DimensionNet project folder. If the files have different filenames to the DimensionNet project, you can choose to rename the files as you upload them.

In most cases, renaming files has no effect on the way those files can be used in DimensionNet, because the files do not normally contain their names or the names of other files. The exception is files to do with quotas: these are typically the .mqd file that contains the project's quota definitions and the *_sm.mdd file that is created when a project's quotas are based on variables in the sample records. If you are uploading files for a project that uses Quota Control, you must be very careful about renaming files.

The .mqd file contains the name of the questionnaire definition (.mdd) file that it relates to. It also contains the name of the *_sm.mdd file if there is one. When you upload these files, the Files activity can rename them but it cannot change the filenames that are stored in the .mqd file, and this name mismatch will cause activation in DimensionNet to fail.

The easiest (and recommended) way to avoid this problem is not to mix DimensionNet and desktop activities during the script creation and activation stages of a project. If you create a questionnaire script using a text editor, you should activate it using the scriptwriter's desktop program or the command-line activate program. If you create a questionnaire script using the Build activity in DimensionNet, you should activate it in DimensionNet too. Do not create questionnaires using desktop programs and then activate in DimensionNet, or vice versa. This will ensure that all files are created with the appropriate names, and that those names appear correctly in files that need to store project filenames.

When a DimensionNet project uses Quota Control, the recommended procedure for creating the .mqd file and making it available to DimensionNet is as follows:

► Create the questionnaire script using Build or mrStudio.

► Download the .mdd file onto your local machine.

► Run Quota Setup to create the .mqd and, if appropriate, the *_sm.mdd files using the downloaded .mdd file.

► Upload the .mqd and *_sm.mdd files into the DimensionNet project folder. Since the filenames already match the project name no renaming will be necessary.
