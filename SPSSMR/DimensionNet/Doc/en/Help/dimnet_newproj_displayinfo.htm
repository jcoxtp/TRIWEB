Defining Display Settings for Mobile Projects

You can define how a project is displayed on a remote device by providing a displayinfo.xml file with the project. The displayinfo.xml file should be in the same directory as the .mdd file when the project is activated. The following is an example file:

<?xml version="1.0" encoding="UTF-8"?> 
<Project>
<Description>Test Survey</Description> 
  	<ExpiryDate>2008-05-04</ExpiryDate> 
  	<Interview Datasource="mrDataFileDsc" Routing="Paper" Renderer="CATIPlayer"/> 
 	<CaselistView>
 		<Fields>
  			<Field Name="Respondent.Serial"/> 
  			<Field Name="Respondent.ID"/> 
				<Field Name="DataCollection.Status"/> 
  		</Fields>
  </CaselistView>
</Project>
EntryDescription
DescriptionDisplayed in the project list on the remote device
ExpiryDateNot implemented
Interview DatasourceOnly mrDataFileDsc is valid at this time
RoutingIndicates which Routing context to use on the remote device
RenderedIndicates which player to use. Valid values are CATIPlayer and HTMLPlayer.
FieldsList of the VDATA names of the fields you want listed in the remote device’s case list. This must include the following fields:

• Respondent.Serial

• Respondent.ID

• DataCollection.Status

Note: It is recommended that the fields have short labels in the Analysis context.
