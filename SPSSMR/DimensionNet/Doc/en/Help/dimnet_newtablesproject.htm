Creating a New Analysis Project

This section covers creating a non-mrInterview project.

Before you set up a non-mrInterview project, you need to identify the data files you are going to use and, if necessary, prepare them for uploading. See Preparing Your Data Files For Uploading for more information.

â–º Setting up a project involves three main steps:

• Create the new project in DimensionNet

• Check the uploaded files into the Shared folder

• Create and populate a test table

After you have followed these steps, your project will be available to all users who belong to the roles you selected as having access to the project.

Tip: Sample data is included with the Dimensions Development Library, which is available as a separate installation and as a free download. The samples include a backed-up relational MR (RDB) database and associated Metadata Document (.mdd) file, Quanvert databases, Dimensions XML data sets, SPSS .sav, and Quancept .qdi and .drs files. The Dimensions Development Library provides detailed step-by-step instructions for setting up DimensionNet projects for each of the main sample data sets.
