The DimensionNet Screen

The DimensionNet screen is divided into two sections. On the left is the project list, showing the projects you may work on. On the right is the activity list for the project you have chosen.

The Project List

The project list displays all projects that exist in DPM and that you can access. An icon in front of each project name tell you about the project's status.

Inactive. The project is not yet available for interviewing, or is a project that is for use in mrTables only.
Test. The project can be used for running test interviews.
Active. The project can be used for live interviewing.
Inactive and locked.
Test and locked.
Active and locked.

If you hover over an icon or a project name, DimensionNet displays a pop-up showing the project's type (mrInterview or mrTables) and status. If the project is locked, the pop-up tells you who has locked it.

For further information about the project list see The Project List.

The Activity Information Panel

Whenever an activity starts, the Activity Information Panel is displayed in the upper right portion of the screen. It lists the name, ID and description of the current project and provides brief help about what to do next. You can shrink the panel so that only the title bar is visible by clicking the button on the right of the title bar. Clicking the button again reopens the panel.

The Activity List

The activities that are available for a project depend on which products are available, what you are allowed to access, whether the project is locked by another user and, for an mrInterview project, the project's status and requirements:

• You will not see mrTables in the list of activities if it has not been installed. Similarly, the mrInterview activities will not be shown if mrInterview has not been installed.

• Access to DimensionNet activities is controlled by roles and each role has access to a different set of projects and activities. Some users may have one role and others several roles. How the roles are set up will vary. For example, users who belong to the ScriptWriterGroupA role might have access to a different set of projects and activities from a user who belongs to the AnalysisGroupB role. The ProjectManagerGroupA role might have access to the same activities but a different set of projects from the ProjectManagerGroupB, and so on.

• When a project is locked by another user, some activities that you can normally access may not be available.

• If an mrInterview project has not been activated, you will see the activities related to setting up a project, including Build, Launch, Participants, and Files, but not those associated with viewing or exporting data. Similarly, if an mrInterview project has been activated but does not use Sample Management or Quota Control, there may be a link for the Survey Results activity so that you can view data for the project, but Quotas or Participant Rules will not be displayed because they are not relevant to this project.
