Locking and Unlocking Projects

DimensionNet automatically locks a project when you use an activity that can change the project files. When a project is locked, everyone except the person who locked the project has restricted access to the project, and most activities will be unavailable to them. This is an important safety mechanism designed to prevent one user accidentally overwriting another user's files.

DimensionNet does not automatically unlock a project when you close the activity that locked it. If you are working on a project that is shared with other users, you should unlock the project when you have finished working on it. Unlocking the project makes it fully available to other users. If you have changed any of the project files, you should use the Files activity to check them in to the shared area so that your changes are visible to other users. Normally you should do this before you unlock the project.

Note that using mrTables does not lock a project, because mrTables never writes data to the main project files. (When you create new variables in mrTables, they are stored separately from the main project data.) This means that you can use mrTables on a project that is locked.

To Lock a Project:

► Select the project in the project list.

► Click the Lock Project icon.

To Unlock a Project:

► Select the project in the DimensionNet project list.

► Click the Unlock Project icon.
